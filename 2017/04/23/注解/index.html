注解

发表于 2017-04-23   |   分类于 Spring

​ 注解可以被用在包，类，方法，变量，参数上，被注解的代码并不会直接被注解影响。这只会向第三系统提供关于自己的信息以用于不同的需求。注解会被编译至class文件中，而且会在运行时被处理程序提取出来用于业务逻辑。以下是开发中使用到的一些注解，可能描述存在不规范的情况，后续会继续添加和修改。

Spring注解

​ 在springmvc框架中使用，作用和xml文件中的Bean一样，需要在 XML 配置文件中启用Bean 的自动扫描功能，这可以通过实现，Spring 在容器初始化时将自动扫描 base-package 指定的包及其子包下的所有 class文件。使用注解后，就不再需要在 XML 中显式使用 进行Bean 的配置。

1.@Controller

​ controller层使用。添加注解@Controller后表示该类为servlert，可以同时存放多个servlert方法在这个类中。

2.@Service

​ service实现类层使用。例如：@Service(“userservice”)，可以对 controller层属性中的对应接口赋值。

3.@Repository

​ dao层实现类使用。

4.@Componment

​ 如果在这个类即不是Controller,又不是Service又不是Dao,v那么就顶componment;Util(工具类); 它是父类,Repository,service,Controller是子类

5.@Resource

​ 作用相当于xml里面的property，可以为类的属性赋值。

ssm框架注解

1.@RequestMapping

​ RequestMapping注解是一个用来处理请求地址映射的注解，可用于类或方法。用于类上，表示类中的所有响应请求的方法都是以该地址作为父路径。访问路径为：url//content/cat/getTree

2.@PathVariable

@RequestMapping("{param}")
public String page(@PathVariable("param") String jumPage){
    System.out.println(jumPage);
    return "goods/goods-"+jumPage;
}

可以获取 RequestMapping中的请求参数,可以用于公共页面的请求跳转。

3.@Value

可以获取属性文件中的值，比如：此处代码中的文件上传的地址用户参数值存放于fileupload.properties文件中，需要在配置文件中加载文件路径

4.@ResponseBody

用于表示直接返回信息到jsp页面。

5.@Autowired

自动装配

需要在配置文件中添加扫描

6.@RequestParam

表示jsp页面发送到controller层的请求参数，可以设置默认值。
