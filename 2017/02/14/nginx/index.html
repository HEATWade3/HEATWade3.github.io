Nginx

发表于 2017-02-14   |   分类于 服务器

​ nginx是用于反向代理的服务器，它是一个程序转发请求，是一个网站服务。

简述Nginx的作用

​ 程序员开发的服务上线,放到linux系统上面,服务器要7*24小时,不能关机,服务永不宕机，如果开发的服务,放到linux里面的tomcat,升级服务先停掉tomcat,将写的程序web服务重新放到webapps中,启动tomcat。升级过程中tomcat停止的时间这期间用户无法访问服务,这样肯定会影响用户的使用和体验。

​ 所以，需要系统升级的时候得保证用户使用服务不宕机,服务器维护或故障的时候，停止tomcat不能让用户感觉到。使用nginx后可以做到这一点，需要配置多个服务器，每个服务器中放置相同的项目。用户访问时先访问ngin，由nginx分配决定去访问哪一个服务器，同时，当多个服务器某个出现异常情况时，nginx不会转发用户请求到该服务器。

​ 这样，nginx可以降低单个服务器的负载，降低高并发，保证Linux系统上的服务器端能一直工作（即使是出现维护、故障的情况）。

  1. 让配置文件生效

​ Service nginx restart(重启)

​ nginx -t(检查一下配置文件是否有错),nginx -s reload(重新加载配置文件的内容)

​ 配置文件中的root目录和地址/file可以修改；

​ 为了配合下载站,如果显示目录不存在需要创建，同时在显示目录/data/savefile下面创建一个file目录，用来存

3.访问

浏览器访问url地址为：nginx服务器IP/file/

会显示出file目录下的所有文件，点击的时候,浏览器会尝试打开此类型的文件,如果能够打开,直接就打开,如果打不开,就弹出一个下载框,让你下载。

创建用户：useradd ftpuser
